5. FEDERATED LEARNING FLOW
================================================================================

ROUND r:

1. COORDINATOR distributes:
   - Current global model weights
   - Round number r
   - Dataset partition indices for each office

2. EACH OFFICE i (in parallel):
   a. Load local NSL-KDD partition
   b. Train XGBoost locally for 5 epochs
   c. Extract model weight vector W_i
   d. Apply Laplace DP: W_i' = W_i + noise(epsilon=0.5)
   e. Send W_i' to coordinator
   f. Send identity hash to blockchain (commit phase)

3. COORDINATOR receives all W_i':
   a. Run Krum aggregation on {W_0', ..., W_n'}
   b. Krum identifies and rejects Byzantine weights
   c. Aggregate remaining weights: W_global = mean(W_honest)
   d. Update global model

4. BLOCKCHAIN UPDATE:
   a. For each rejected update:
      - slash reputation by 50 points
      - log slashing event with timestamp
   b. Offices below reputation threshold (e.g., 50) are BANNED

5. EVALUATION:
   a. Test global model on held-out test set
   b. Compute: accuracy, precision, recall, F1
   c. Log all metrics to round_metrics.csv

6. REPEAT until convergence or max rounds

6. EXPERIMENTS TO RUN
================================================================================

EXPERIMENT 1: BASELINE (All Honest)
-----------------------------------
Objective: Establish baseline performance
Setup: 5 offices, all train normally, no attacks
Expected: Accuracy increases each round, Krum selects any weights
Metrics: Global accuracy, per-office accuracy, training loss

EXPERIMENT 2: SINGLE BYZANTINE ATTACK
--------------------------------------
Objective: Verify Krum detects poisoning
Setup: Office 4 sends [10.0, 10.0, 10.0, 10.0] as weights
Expected: Krum assigns high score to office 4, rejects weights
Metrics: Krum scores, rejection rate, accuracy impact

EXPERIMENT 3: MULTI-BYZANTINE (2 Offices Attack)
-------------------------------------------------
Objective: Test Krum threshold with f=2 malicious
Setup: Office 3 and 4 both poison
Expected: Krum should still identify outliers
Metrics: Detection accuracy, false positive rate

EXPERIMENT 4: DIFFERENTIAL PRIVACY SENSITIVITY
-----------------------------------------------
Objective: Tradeoff between privacy and accuracy
Setup: Vary epsilon = [0.1, 0.3, 0.5, 1.0, 2.0]
Expected: Lower epsilon = more privacy, slightly lower accuracy
Metrics: Accuracy vs epsilon curve, privacy budget consumed

EXPERIMENT 5: REPUTATION SLASHING DYNAMICS
-------------------------------------------
Objective: Show reputation recovery is possible
Setup: Attack for 3 rounds, then honest for 5 rounds
Expected: Reputation drops on attack, recovers slowly after
Metrics: Reputation over rounds, banned nodes count

EXPERIMENT 6: ABLATION STUDY
-----------------------------
Objective: Isolate contribution of each component
Variants:
- No Krum (just average): expected drop in attack scenarios
- No DP: expected privacy leak
- No Blockchain: expected no reputation tracking
